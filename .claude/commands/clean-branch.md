マージ済みのローカルブランチを整理（削除）する。

## 手順

1. ベースブランチを特定する（優先順: `develop` → `main`。リポジトリに存在するものを使う）
2. ベースブランチに `git switch` で移動する（既にそのブランチにいる場合はスキップ）
3. `git fetch --prune` でリモートの最新状態を取得し、削除済みリモートブランチの追跡を除去する
4. `git branch --merged <base>` でベースにマージ済みのローカルブランチ一覧を取得する（`main` / `develop` および現在チェックアウト中のブランチは除外）
5. 削除対象のブランチ一覧をユーザーに表示し、削除してよいか確認する
6. ユーザーが承認したら `git branch -d <branch>` で各ブランチを削除する
7. 削除結果を表示する

## ルール

- `main` / `develop` ブランチは絶対に削除しない
- 現在チェックアウト中のブランチは削除対象から除外する
- マージ済みブランチのみを対象とする（`-d` を使い、`-D` は使わない）
- 削除前に必ずユーザーの確認を取る
